# Algorzm AI Operating System

This repository is the operating system for Algorzm's AI-assisted problem solving.

It is not a Codex configuration repository.

It is a technology-agnostic, continuously evolving leverage system for consulting, product building, software engineering, research, automation, visualization, presentation, and AI agent coordination.

## Purpose

The goal is to make every AI agent work with Algorzm's problem-solving philosophy while reducing executive friction.

AI agents should not merely produce code.

They should:

1. understand the real problem
2. think strategically
3. research before building
4. evaluate tools, MCP servers, libraries, dashboards, multimodal options, and automation opportunities
5. avoid shallow vibe coding
6. convert CEO intent into structured execution
7. preserve reusable learning by improving the operating system
8. use PRs, issues, and Mission Control flows instead of forcing the CEO into developer workflows

## Core Idea

Every interaction should create reusable intelligence.

CEO intent becomes structured missions.

Missions become issues, PRs, playbooks, skills, or operating-system improvements.

Feedback is not merely archived.

Important feedback is absorbed into the system.

## Repository Structure

- `CONSTITUTION.md`  
  Highest-level philosophy and permanent principles.

- `ARCHITECTURE.md`  
  Hierarchy, authority levels, lifecycle, and system design.

- `AGENTS.md`  
  Runtime behavior rules for AI agents.

- `governance/`  
  Rules for how principles, technologies, tools, and workflows are evaluated and promoted.

- `mission-control/`  
  CEO-facing automation model, permission model, and remote-control architecture for agentic execution.

- `.github/ISSUE_TEMPLATE/`  
  Structured mission intake templates.

- `.github/workflows/`  
  Automation scaffolding for future agentic runners.

- `rules/`  
  Validated reusable operating rules.

- `playbooks/`  
  Repeatable workflows for recurring work.

- `skills/`  
  Portable packaged capabilities.

- `memory/`  
  Distilled long-term preferences, mistakes, patterns, and lessons.

- `feedback/`  
  Raw feedback only when it cannot yet be absorbed into existing system documents.

## CEO Role

The CEO owns intent, strategy, priority, taste, and judgment.

The CEO should not be forced to manage Git, branches, commits, PRs, tooling, or implementation details.

## AI Role

AI agents execute, reason, research, build, review, record, and improve the operating system.

## CTO Role

The CTO agent translates CEO intent into technical execution, maintains the operating system, evaluates leverage opportunities, and reduces CEO friction.

## Golden Rule

Do not leave important knowledge trapped inside chat history.

If something was learned, absorb it into the system layer where it belongs.
