# 정성제안 증강 트랙 헌장 v1
승인 계보: CEO 2026-07-09 "정성제안서는 아예 다른 어프로치" + 발표브리핑 극찬 → 그 원형의 공식화. 자동화 금지 — 증강만.

## 원칙
정량은 기계가 채우고(paperwork), 정성은 사람이 이긴다. 이 트랙에서 AI는 대필가가 아니라 **참모**다: 신호를 캐고, 무기를 대령하고, 반론을 때려보고, 장표를 찍는다. 논증과 설득의 주연은 CEO.

## 증강 5요소 (창업진흥원 R26BK01596499에서 검증된 원형)
1. **RFP 신호 추출** — 배점 급소(등급제 계단, 배점 큰 항목), RFP가 흘린 힌트(반복 강조 문구, 명시된 산출물, 첨부 참고자료), 평가 일정·장소. 산출: 발표브리핑 1장.
2. **지식베이스 검색** — deck-index 낱장 인덱스에서 "이 논증엔 과거 이 장표"(프레임워크·도메인 태그 매칭) + 유사실적 자동 매칭(v10 엔진). 산출: 재사용 장표 후보 목록.
3. **구조 초안 제안** — RFP 목차를 1:1 미러링한 제안서/발표 골격 + 항목별 공략 포인트. 콘텐츠는 채우지 않는다(사람 몫).
4. **레드팀** — 심사위원 페르소나(내부 실무·외부 교수)로 예상 공격 질문 + 응수 라인. 발표 D-2 리허설 상대역.
5. **장표 생산** — 확정된 논증을 slide-forge(스펙 JSON→PPTX)로 렌더. 밀도 프리미티브(ctx.table 등) 사용.

## 게이트
- 모든 산출물은 제안(draft) 신분 — CEO 채택 전엔 제출물 아님
- 수치는 출처 필수, 없으면 (가정) — slide-forge 특칙 상속
- 검증되지 않은 성과 수치·과장 문구 금지 (홈페이지 forbidden_claims 철학 공유)

## 트리거
새 강추 공고의 딥분석 완료 시 1요소(신호 추출)는 자동 생성 후보 — 나머지는 CEO가 "이 건 들어간다" 선언 시 가동.
