# CTO Review Mode
Version: 0.1

## Purpose

CTO Review Mode prevents an agent from jumping directly into implementation after a shift reset, context reset, repo handoff, or major architecture request.

The first responsibility is synchronization, critique, and design review.

## When To Use

Use CTO Review Mode when:

- the agent is rejoining after a shift reset
- the repository or operating system changed
- the CEO asks for planning, audit, strategy, or architecture
- a task may affect multiple projects or governance layers
- the implementation path is unclear
- the CEO has recently corrected process or direction

## Required Read Order

1. `README.md`
2. `CONSTITUTION.md`
3. `ARCHITECTURE.md`
4. `AGENTS.md`
5. `governance/`
6. `mission-control/`
7. relevant `brain/`
8. relevant project files

## Review Checklist

The agent must evaluate:

1. contradictions
2. duplicated concepts
3. outdated references
4. missing layers
5. unnecessary documents
6. missing governance
7. unabsorbed CEO feedback
8. tool-specific assumptions
9. CEO friction
10. agentic execution quality
11. Mission Control quality
12. Shift Management quality
13. Cross-Agent Handoff quality
14. automatic execution opportunities
15. automatic approval routing opportunities
16. future leverage from MCP, multimodal input, dashboards, orchestration, evaluation, context engineering, and new AI capabilities

## Required Output

Before modifying files, produce an Architecture Review Report with:

- Critical issues
- Important issues
- Minor improvements
- Missing capabilities
- Recommended repository refactoring
- Recommended next sprint

## Rule

Do not implement during CTO Review Mode unless the CEO explicitly says to proceed.
