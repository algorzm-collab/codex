# CEO Profile
Version: 1.0 (2026-07-09, CTO 작성 — 대화 관찰 기반. CEO 검수 대기)
용도: 모든 에이전트가 CEO를 이해하고 받치기 위한 공유 컨텍스트. 아첨 금지, 관찰만 기록.

## 정체
- 전략 컨설팅사 대표. 비개발자이나 시스템 사고는 개발자 이상 (자작 AI OS 헌법 보유).
- 도메인: 공공부문 전략·조직·보수체계·창업지원 정책. 실증 고객군: 에너지(가스기술공사·한전원자력연료), 창업지원(TIPS/DIPS 계열), 농축산(축산물품질평가원), 기술(한국광기술원) 등.
- 프레임워크를 원어민처럼 구사: BCG 매트릭스, Porter 가치사슬, TRL/Stage-Gate, Real Options, Lean Startup, Futures Wheel, Pay-Band 설계. 출처 표기 습관 있음 ("Source: ... 기반 재구성").

## 작업 수준 (품질 기준선)
- 산출물은 발표자료가 아니라 **고구조화 보고서** (A4 밀도 극대화).
- 핵심 기준: **"기하학이 논증하는가"** — 요소의 위치·중첩·화살표가 주장을 수행해야 함. 박스 나열 = 실격.
- 본인이 세운 품질 서열: 인턴 < 일반 ChatGPT < 슬라이드툴 < AI 그리드형 < 본인 이미지엔진 산출 < 본인 수작업.
- 할루시네이션 = 즉시 실격. 단 출처 명기된 가정(Assumption) 기반 추론은 정당한 방법론으로 인정.

## 의사결정 스타일
- 투자자형: 킬 기준(3회 반복), 손실 상한, 조건부 베팅을 스스로 설정.
- **복종이 아니라 비판을 요구** — "말 잘 듣는 지원이 아니라 CTO가 필요해." 무비판 수용은 신뢰를 깎음. 최선이면 근거와 함께 수용, 아니면 비판+대안 제시가 의무.
- 증거로 설득됨. 감정이 격해도 논리 구조(비용·대안·킬기준)를 제시하면 수용함.
- 돈보다 시간·신뢰 낭비에 분노. 실패 자체보다 검증 없는 출고에 분노.

- 진행 나레이션 싫어함 (Cave Mode). 권한 프롬프트·마찰 극도로 싫어함 (턴키 요구).

## 에이전트 행동 지침 (이 프로파일의 실전 적용)
1. 산출물은 자체 QA 통과 전 CEO에게 보이지 말 것 — 검증 없는 출고가 신뢰를 가장 크게 훼손
2. 비판 요청 시 진짜로 비판할 것 — 약점 회피는 들킴
3. 실패 보고는 원인·비용·다음 행동과 함께 즉시
4. CEO의 시간을 쓰는 요청(채점·승인)은 최소화하고 묶어서
