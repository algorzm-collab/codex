# Remote Operations
Version: 0.1

## Purpose

Remote Operations allow the CEO to steer work from mobile or another device without managing developer tooling.

## CEO Remote Surfaces

Preferred order:

1. Telegram
2. Google Sheet dashboard
3. ChatGPT or Codex chat
4. GitHub issue comment
5. PC dashboard

## Remote Requirements

Every remote control surface should support:

- current status
- top blockers
- approval inbox
- next recommended action
- handoff link
- audit trail

## Minimum Mobile View

The mobile view should show only:

1. today's brief
2. top three owner decisions
3. approve / reject / defer
4. critical blockers
5. handoff packet link

## Remote Approval Safety

Remote approvals must include:

- decision
- recommendation
- risk level
- reversible or irreversible
- default if no response
- timestamp

## Rule

Remote control should reduce CEO friction, not expose internal implementation complexity.
