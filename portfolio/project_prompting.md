# Project Prompting Operating Model

## Purpose

This document defines how prompting works when the CEO manages many projects through ChatGPT.

The CEO should not create a different complex prompt for every project.

The system should make project context explicit, reusable, and routed.

---

## Core Rule

The CEO uses one simple command style.

The system applies project-specific context behind the scenes.

```text
[project name] + [intent]
```

Examples:

```text
G2B자동화 다음 단계 진행해
장표머신 이번 주 산출물 정리해
개인 브랜딩 콘텐츠 구축 계속해
Strat.kr 리스크 리뷰해
```

---

## Project Prompt Stack

Every project should have a four-layer prompt stack.

```text
CEO Intent
  ↓
Project Context
  ↓
AI OS Principles
  ↓
Execution Task
```

### 1. CEO Intent

The CEO says what should happen.

Short is better.

### 2. Project Context

Stored in ChatGPT Project, portfolio registry, files, or GitHub.

Contains:

- project goal
- business context
- current status
- key constraints
- repo link
- active issue / mission
- tone / quality preferences

### 3. AI OS Principles

Always inherited from:

- `CONSTITUTION.md`
- `ARCHITECTURE.md`
- `AGENTS.md`
- `governance/`
- `mission-control/`

### 4. Execution Task

Generated by ChatGPT CTO.

May become:

- direct answer
- GitHub issue
- Codex micro task
- research task
- deck task
- dashboard task
- decision request

## CEO Prompt Template

The CEO should usually say:

```text
[project] [what you want]
```

Optional modifiers:

```text
빠르게
깊게
비판적으로
사업성 위주로
코덱스 마이크로태스크로
장표로
대시보드로
```

---

## CTO Behavior

When the CEO gives a project-level instruction, ChatGPT CTO should:

1. identify the project
2. load or infer project context
3. check portfolio registry
4. decide whether action belongs in ChatGPT, GitHub, Codex, Library, Apps, or external dashboard
5. decompose if needed
6. execute directly where possible
7. create Codex micro task only when implementation is needed
8. report progress, risk, and next action

---

## Codex Prompting Rule

The CEO should not manually design Codex prompts.

ChatGPT CTO writes Codex prompts.

Codex receives only:

- one repo
- one issue or micro task
- one narrow scope
- one stop condition

---

## Project Prompt Ownership

ChatGPT Project owns:

- ongoing CEO conversation
- project-specific business context
- current decision history

GitHub owns:

- code
- issues
- pull requests
- implementation record

Library owns:

- reusable assets
- templates
- prompts
- playbooks
- examples

Codex owns:

- bounded implementation
- verification
- small commits or concise reports

---

## Golden Rule

The CEO should never ask: which prompt do I use for this project?

The CEO states intent.

The CTO layer routes and prompts the right system.
