# STRATEGY Sprint 04: Admin Analytics Summary 기획안 v1

작성일: 2026-07-01

## 1. 목적

Sprint 03에서 이벤트 저장은 가능해졌다. 그러나 이벤트 목록만으로는 대표자가 빠르게 판단하기 어렵다.

Sprint 04의 목적은 관리자 콘솔에서 다음 질문에 즉시 답하게 만드는 것이다.

```text
오늘/최근 방문자는 어떤 행동을 했는가?
어떤 진단 유형이 많이 나오는가?
어떤 대표실적 proof가 클릭되는가?
문의 전환 직전 행동은 무엇인가?
```

## 2. 1차 구현 범위

관리자 API `/api/admin/records`에 `event_summary`를 추가한다.

요약 항목:

- 전체 이벤트 수
- 이벤트명별 카운트
- 진단 유형별 카운트
- 대표실적/타깃별 카운트
- 최근 이벤트 200개

관리자 화면:

- `이벤트` 탭 상단에 요약 카드 표시
- 그 아래 최근 이벤트 목록 표시

## 3. 성공 기준

- 관리자 로그인 후 이벤트 탭에서 요약이 보인다.
- `issue_result_view`, `proof_jump_click`, `contact_form_submit` 등이 구분된다.
- 대표실적 클릭 대상이 보인다.
- 기존 관리자 기능을 깨지 않는다.
