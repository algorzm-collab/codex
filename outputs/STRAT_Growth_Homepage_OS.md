# STRAT Growth Homepage OS

## 1. Direction

STRAT 홈페이지는 전통적인 컨설팅 회사 소개 페이지가 아니라, 방문자의 문제를 진단하고 상담 전환 데이터를 축적하는 그로스 엔진으로 설계한다.

기존 컨설팅 홈페이지의 흐름:

`회사 소개 -> 서비스 소개 -> 실적 -> 전문가 -> 문의`

STRAT Growth Homepage의 흐름:

`문제 자각 -> 1분 진단 -> 맞춤 인사이트 -> 신뢰 증명 -> 낮은 부담의 CTA -> 행동 데이터 축적 -> 카피/콘텐츠/CTA 개선`

핵심 전환:

- 홈페이지를 브로슈어가 아니라 제품처럼 설계한다.
- 첫 화면에서 STRAT을 설명하지 않고 방문자의 문제를 선택하게 한다.
- 방문자 선택값에 따라 메시지, 사례, CTA를 달리 보여준다.
- 문의 전에도 작은 가치를 제공한다.
- 모든 섹션과 CTA는 측정 가능한 실험 단위가 된다.

## 2. Strategic Thesis

STRAT의 홈페이지는 다음 한 문장으로 설계한다.

> 공공기관과 기업의 복잡한 전략 과제를 1분 안에 진단하고, 실행 가능한 컨설팅 방향으로 연결하는 Growth Advisory Platform.

브랜드 메시지는 다음 방향을 가진다.

- 전통 컨설팅의 신뢰
- 스타트업식 속도
- 제품형 UX
- 데이터 기반 개선
- 공공/기업 전략 과제에 대한 깊은 문제 이해

## 3. Growth Flywheel

### 3.1 Attract

방문자는 다음 문제 키워드로 유입된다.

- 중장기전략
- 조직진단
- 경영평가
- 성과관리
- 인사/조직 혁신
- ESG
- AI/DT 전환
- 공공기관 혁신계획
- 사업전략
- 제안/입찰 전략

유입 자산:

- 문제별 랜딩페이지
- 공공기관 전략진단 콘텐츠
- 체크리스트형 리드마그넷
- 사례 기반 SEO 아티클
- LinkedIn/브런치/뉴스레터 콘텐츠
- 검색광고용 고의도 키워드 페이지

### 3.2 Activate

첫 방문자가 10초 안에 해야 할 행동:

- 자신의 조직 유형 선택
- 현재 과제 선택
- 긴급도 선택
- 원하는 산출물 선택

예시:

`공공기관 / 중장기전략 / 3개월 내 착수 / 보고서+워크숍 필요`

Activation의 목적은 방문자에게 “이 회사가 내 문제를 구조화해준다”는 첫 경험을 주는 것이다.

### 3.3 Engage

선택값에 따라 다른 콘텐츠를 보여준다.

공공기관을 선택한 경우:

- 경영평가 대응
- 기관 혁신계획
- 중장기 경영전략
- 조직/인력 진단
- 성과지표 설계

민간기업을 선택한 경우:

- 성장전략
- 신사업전략
- 시장진입전략
- 조직운영체계
- AI/DT 전환

방문자는 모든 서비스를 읽는 것이 아니라, 자기 상황에 맞는 흐름만 본다.

### 3.4 Convert

문의 CTA는 추상적인 “문의하기”가 아니라 문제 기반으로 설계한다.

권장 CTA:

- 전략 과제 1분 진단하기
- 우리 기관에 맞는 컨설팅 유형 찾기
- 제안 가능성 검토받기
- 30분 전략상담 예약
- 경영평가 대응 방향 받기
- 중장기전략 착수 가능성 확인하기

전환 폼은 짧게 시작하고, 이후 단계에서 추가 정보를 받는다.

1단계:

- 조직 유형
- 과제 유형
- 착수 희망 시점
- 이름/연락처

2단계:

- 현재 상황
- 예산 범위
- 내부 의사결정 구조
- 참고자료 업로드

### 3.5 Learn

홈페이지는 다음 데이터를 학습해야 한다.

- 어떤 유입 채널이 상담으로 이어지는가
- 어떤 과제 선택이 많은가
- 어떤 CTA가 가장 많이 클릭되는가
- 어느 섹션에서 이탈하는가
- 문의 전환자는 어떤 콘텐츠를 봤는가
- 공공기관/민간기업별 전환 경로가 어떻게 다른가

### 3.6 Amplify

상담과 프로젝트에서 나온 패턴을 콘텐츠화한다.

- “공공기관 중장기전략 착수 전 체크리스트”
- “경영평가 대응에서 자주 놓치는 7가지”
- “조직진단 프로젝트가 실패하는 이유”
- “AI 전환 전략 수립 전 반드시 정리할 질문”
- “전략보고서가 실행으로 이어지지 않는 이유”

이 콘텐츠가 다시 검색 유입과 신뢰 형성을 만든다.

## 4. Homepage UX Architecture

### 4.1 Hero

목표:

방문자가 첫 화면에서 자기 문제를 선택하게 한다.

권장 구조:

- 상단: STRAT 로고, Services, Cases, Insights, About, Contact
- 메인 카피: 문제 진단형
- 서브 카피: STRAT의 전문성
- 인터랙티브 진단 위젯
- 신뢰 지표

Hero 카피 후보:

1. 지금 필요한 전략 과제를 1분 안에 진단하세요.
2. 공공기관과 기업의 복잡한 경영문제를 실행 가능한 전략안으로 바꿉니다.
3. 중장기전략, 조직진단, 경영평가, ESG, AI전환. 지금 필요한 컨설팅 방향부터 찾으세요.
4. 컨설팅을 의뢰하기 전, 먼저 우리 조직의 전략 과제를 구조화하세요.
5. STRAT은 전략 과제를 진단하고, 설계하고, 실행 가능한 의사결정안으로 만듭니다.

Hero 위젯:

질문 1. 어떤 조직인가요?

- 공공기관
- 지자체/출자출연기관
- 민간기업
- 스타트업/신사업조직

질문 2. 현재 과제는 무엇인가요?

- 중장기전략
- 조직진단
- 경영평가
- 성과관리
- ESG
- AI/DT
- 신사업/성장전략
- 제안/입찰전략

질문 3. 언제 필요하신가요?

- 즉시
- 1개월 내
- 3개월 내
- 검토 중

결과:

- 추천 컨설팅 유형
- 예상 착수 프로세스
- 관련 사례
- 상담 CTA

### 4.2 Problem Mirror

방문자의 상황을 그대로 비춘다.

섹션 카피:

> 이런 상황이라면 STRAT의 전략진단이 필요합니다.

문제 카드:

- 전략보고서는 있는데 실행 우선순위가 불명확하다.
- 경영평가 대응이 매년 반복 대응에 머문다.
- 조직개편이 필요하지만 내부 설득 논리가 부족하다.
- ESG/AI 전환 과제가 있지만 실행 로드맵이 없다.
- 신규사업 방향은 있지만 시장성과 사업성이 정리되지 않았다.
- 용역 발주 전 과업 범위와 산출물을 명확히 잡고 싶다.

### 4.3 Diagnostic Engine

홈페이지의 핵심 제품이다.

이름 후보:

- STRAT Fit Finder
- Strategy Fit Check
- 1-Minute Strategy Scan
- STRAT Growth Scan
- Public Strategy Navigator

기능:

- 방문자 유형 분류
- 과제 유형 분류
- 긴급도 분류
- 추천 서비스 매칭
- 관련 사례 추천
- 상담 CTA 생성

진단 결과 예시:

> 현재 선택하신 과제는 “공공기관 중장기전략 + 경영평가 연계형”에 가깝습니다.  
> 초기 단계에서는 환경분석보다 기관의 핵심성과지표, 정부정책 연계성, 내부 실행체계 점검이 먼저 필요합니다.

### 4.4 Proof Layer

실적은 단순 리스트보다 문제-접근-산출물 구조로 보여준다.

사례 카드 구조:

- 고객 유형
- 문제
- STRAT 접근
- 핵심 산출물
- 결과/효과
- 관련 서비스

예시:

고객 유형: 공공기관  
문제: 중장기전략과 경영평가 지표의 단절  
접근: 정책환경 분석, 기관 역할 재정의, 성과체계 재설계  
산출물: 중장기 전략체계도, 실행과제, KPI 맵, 로드맵  
CTA: 유사 과제 상담받기

### 4.5 Method Layer

방법론은 추상적 철학보다 산출물 중심으로 보여준다.

STRAT Method:

1. Issue Framing
2. Evidence Mapping
3. Strategy Design
4. Operating Model
5. Execution Roadmap
6. Decision Package

각 단계별로 고객이 받는 산출물을 명시한다.

- 이슈맵
- 이해관계자 분석
- 정책/시장/경쟁 분석
- 전략체계도
- 실행과제
- KPI 체계
- 조직/인력 개선안
- 최종보고서
- 발표자료
- 내부 의사결정 패키지

### 4.6 Conversion Layer

하단 CTA는 한 번 더 진단형으로 배치한다.

카피:

> 아직 과업 범위가 명확하지 않아도 괜찮습니다.  
> 현재 상황을 선택하면 STRAT이 상담 전 검토 방향을 정리합니다.

CTA:

- 1분 전략진단 시작
- 30분 전략상담 예약

문의 폼 문항:

- 조직 유형
- 과제 유형
- 착수 희망 시점
- 현재 고민
- 연락처

## 7. Experiment Backlog

### Experiment 1. Hero CTA

Hypothesis:

진단형 CTA가 일반 문의 CTA보다 클릭률과 문의 전환율이 높다.

Variants:

- A: 문의하기
- B: 1분 전략진단 시작
- C: 우리 조직에 맞는 컨설팅 유형 찾기

Metric:

- CTA click rate
- Diagnostic completion rate
- Lead conversion rate

### Experiment 2. Visitor Segmentation

Hypothesis:

조직 유형을 먼저 선택하게 하면 이후 콘텐츠 체류시간과 문의 전환율이 높아진다.

Variants:

- A: 공통 홈페이지
- B: 공공기관/민간기업 선택형 Hero

Metric:

- First interaction rate
- Scroll depth
- Case section click rate
- Lead conversion rate

### Experiment 3. Proof Format

Hypothesis:

실적 로고/목록보다 문제-접근-산출물 구조의 사례 카드가 신뢰와 전환에 더 효과적이다.

Variants:

- A: 프로젝트 리스트
- B: 사례 카드
- C: 사례 카드 + 산출물 미리보기

Metric:

- Case click rate
- CTA click after case view
- Consultation form start rate

### Experiment 4. Lead Magnet

Hypothesis:

상담 전에 체크리스트를 제공하면 리드 수집률이 높아진다.

Assets:

- 공공기관 중장기전략 착수 체크리스트
- 조직진단 사전 점검표
- 경영평가 대응 체크리스트
- AI전환 전략 준비도 진단표

Metric:

- Download conversion rate
- Email capture rate
- Sales qualified lead rate

### Experiment 5. Form Length

Hypothesis:

짧은 1단계 폼 후 추가 질문을 받는 방식이 전체 문의 완료율을 높인다.

Variants:

- A: 긴 문의 폼
- B: 4문항 짧은 폼
- C: 진단 완료 후 연락처 입력

Metric:

- Form start rate
- Form completion rate
- Qualified lead rate

## 8. Design Principles

UX 원칙:

- 첫 화면은 설명이 아니라 선택이다.
- 추상 메시지보다 문제 언어를 먼저 쓴다.
- 전문성은 사례와 산출물로 증명한다.
- 문의는 마지막 행동이 아니라 진단의 자연스러운 다음 단계가 된다.
- 모든 버튼은 하나의 실험 단위다.

UI 원칙:

- 컨설팅사답게 신뢰감은 유지하되, 스타트업 제품처럼 빠르고 선명하게 작동해야 한다.
- 카드 남발보다 선택형 인터랙션, 데이터 패널, 사례 미리보기, 진단 결과 UI를 중심으로 구성한다.
- 타이포그래피는 강하고 간결하게, 섹션은 짧은 스캔 단위로 나눈다.
- 공공기관 담당자도 부담 없이 신뢰할 수 있는 절제된 톤을 유지한다.
- 혁신적이되 장식적이지 않아야 한다.

- 리드마그넷 다운로드
- 이메일 자동응답
- CRM 연동

- 콘텐츠 추천
- 진단 결과 PDF 생성

3차에서 확장할 것:

- 사용자 세그먼트별 개인화
- 산업별/과제별 케이스 라이브러리
- 컨설팅 준비도 점수화
- 내부 세일즈 대시보드
- SEO 콘텐츠 허브

## 10. Reference Models

참고할 글로벌 UX/성장 모델:

- HubSpot: Flywheel model
- Dave McClure: AARRR pirate metrics
- Reforge/Brian Balfour: Growth loops
