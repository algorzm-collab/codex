# STRAT Website Execution Brief v1

## 1. 한 줄 정의

STRAT 홈페이지는 방문자가 컨설팅 서비스를 읽는 곳이 아니라, 자신의 조직 문제가 전략, 조직, 성과, 인사, ESG, AI/DT 중 어디에 가까운지 직접 좁혀보고 상담으로 이어지는 진단형 그로스 컨설팅 플랫폼이다.

## 2. 핵심 컨셉

제품명:

> STRAT Solution Finder

핵심 경험:

```text
조직 유형 선택
-> 현재 고민 선택
-> 3~5개 핵심 문항 응답
-> 솔루션 후보 1~3개 추천
-> 관련 산출물/사례 확인
-> 진단 결과 기반 상담 신청
```

## 3. 메인 후킹 문장

Primary:

> 지금 조직에 필요한 전략 솔루션을 1분 안에 좁혀보세요.

Problem hook:

> 전략은 있는데 KPI, R&R, 조직구조, 성과평가가 서로 연결되어 있습니까?

Final CTA:

> 아직 과업 범위가 명확하지 않아도 괜찮습니다. 현재 고민을 선택하면 STRAT이 상담 전 검토 방향을 정리해드립니다.

## 6. Solution Finder MVP

### Step 1. 조직 유형

- 공공기관
- 지방공기업/출자출연기관
- 민간기업
- 스타트업/신사업조직
- 아직 정하기 어려움

### Step 2. 현재 고민

- 전략을 새로 수립해야 한다.
- 전략은 있지만 실행이 약하다.
- 조직구조와 R&R이 맞지 않는다.
- KPI와 성과평가가 작동하지 않는다.
- 인사/직무/역량 체계가 필요하다.
- ESG/AI/DT 과제를 실행과제로 바꿔야 한다.

### Step 3. 핵심 문항

선택 경로별 3~5개 문항만 노출한다.

응답 척도:

- 매우 그렇다
- 그렇다
- 보통
- 아니다
- 전혀 아니다
- 잘 모르겠다

중요:

“잘 모르겠다”는 반드시 포함한다. 이는 정보 부족, 내부 공유 부족, 체계 부재를 나타내는 강한 컨설팅 신호다.

### Step 4. 결과

결과 화면:

- 한 줄 진단
- 위험 신호
- 추천 솔루션 1~3개
- 예상 산출물
- 관련 사례
- 상담 CTA

## 12. Decision Needed

다음 의사결정이 필요하다.

1. STRAT Solution Finder를 사이트의 메인 제품처럼 전면 배치할 것인가
2. 첫 화면 카피를 진단형으로 갈 것인가, 브랜드형으로 갈 것인가
3. 초기 타깃을 공공기관 중심으로 갈 것인가, 공공+민간 공통으로 갈 것인가
4. 상담 전환을 “문의”로 부를 것인가, “과제 범위 검토”로 부를 것인가
5. 결과 화면에서 어느 정도까지 무료로 보여줄 것인가

권장 결정:

- Solution Finder 전면 배치
- 진단형 카피
- 공공기관 중심이지만 민간 확장 가능한 구조
- CTA는 “과제 범위 검토받기”
- 결과는 일부 공개 후 상담/이메일 전환
