# STRATEGY Sprint 01: Cinematic Proof System 기획안 v1

작성일: 2026-07-01

## 1. Sprint 목적

STRATEGY 홈페이지의 첫 번째 GitHub식 실험은 대표실적을 단순 목록에서 `증거 장면`으로 바꾸는 것이다.

현재 실적은 많다. 그러나 방문자는 많은 목록을 읽기 전에 먼저 이렇게 느껴야 한다.

```text
이 회사는 공공기관과 기업의 복잡한 전략 문제를 실제로 풀어본 곳이다.
내 문제도 이 경험과 연결될 수 있겠다.
```

따라서 Sprint 01의 목표는 `실적 나열`이 아니라 `신뢰 전환`이다.

## 2. 실험 정의

Issue:

```text
[Experiment] Build cinematic proof panels for six flagship projects
```

Problem:

대표실적이 많지만, 기존 카드형 목록은 신보, 해진공, 주금공, 올림픽 레거시, 도공 미래전략, UAM Team Korea 같은 핵심 실적의 무게감을 충분히 전달하지 못한다.

Hypothesis:

대표실적 6개를 스크롤 기반 cinematic proof panel로 보여주면 방문자는 STRATEGY의 신뢰도를 더 빠르게 이해하고, Issue Finder 이후 문의 CTA로 이동할 가능성이 높아진다.

Primary metric:

```text
trust -> conversion
```

측정 지표:

- case section reach
- flagship panel dwell time
- case-to-contact CTA click
- contact form start
- qualified inquiry count

## 3. 대표 6개 케이스

### 1. 신용보증기금 뉴비전 수립

신뢰 신호:

- 정책금융기관
- 새 비전과 미래 역할 재정의
- 비전-전략방향-실행과제 연결

후킹 문장:

> 정책금융기관의 미래 역할을 다시 정의해야 할 때, 비전은 문장이 아니라 실행체계가 되어야 합니다.

### 2. 해양진흥공사 중장기 경영전략 수립

신뢰 신호:

- 해양산업 정책환경 변화
- 중장기 방향성과 사업 포트폴리오 재정렬
- 전략과 실행 로드맵 연결

후킹 문장:

> 산업 정책이 바뀌면 기관의 역할, 사업 포트폴리오, 실행 우선순위가 함께 재정렬되어야 합니다.

### 3. 한국주택금융공사 부점성과평가 고도화

신뢰 신호:

- 성과관리체계
- 부점 단위 성과평가
- 전략-KPI-현장성과 연결

후킹 문장:

> 성과평가가 구성원을 설득하지 못한다면, 지표가 아니라 전략과 현장의 연결부터 다시 봐야 합니다.

### 4. 강원도개발공사 평창 올림픽 레거시 활용전략

신뢰 신호:

- 올림픽 레거시
- 지역개발
- 중장기 경영전략 및 조직재설계

후킹 문장:

> 대형 이벤트의 유산은 보존 대상이 아니라 지역개발과 기관 전략으로 전환되어야 합니다.

### 5. 한국도로공사 미래사업과제 및 미래도로 전략

신뢰 신호:

- 4차산업혁명
- 드론, 미래도로
- 기술 변화 대응 신규 사업과제

후킹 문장:

> 기술 변화는 유행어가 아니라 기관의 미래사업과 역할을 다시 묻는 질문입니다.

### 6. K-UAM 및 UAM Team Korea 대응전략

신뢰 신호:

- 미래 모빌리티
- UAM 정책/산업 변화
- 기관 대응전략과 실행 우선순위

후킹 문장:

> 새로운 정책 아젠다는 참여 여부가 아니라 우리 기관이 어떤 역할로 들어갈 것인가의 문제입니다.

## 4. UI 원칙

대표실적 섹션은 카드 그리드가 아니다. 스크롤할 때 하나씩 등장하는 전략 장면이어야 한다.

구성:

```text
번호
기관/프로젝트명
문제
STRATEGY 접근
산출물
관련 솔루션
문의 CTA
```

퍼블리싱 원칙:

- 가벼운 HTML/CSS/JS 유지
- CSS scroll animation 활용
- 모바일에서는 세로 스토리 카드로 자연스럽게 전환
- 데이터는 `/api/content`의 상위 대표실적을 재사용
- 별도 라이브러리 추가 금지

## 5. GitHub PR 기준

PR 제목:

```text
feat: add cinematic proof panels for flagship cases
```

PR 체크:

- [ ] 대표 6개 케이스가 상단에 노출된다.
- [ ] 기존 전체 실적 카드 목록은 유지된다.
- [ ] 모바일에서 텍스트가 겹치지 않는다.
- [ ] Issue Finder와 문의 CTA 흐름이 깨지지 않는다.
- [ ] 무거운 라이브러리를 추가하지 않았다.
- [ ] 모션 축소 옵션에서도 정보 접근이 가능하다.
