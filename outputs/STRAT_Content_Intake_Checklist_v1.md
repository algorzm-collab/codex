# STRAT Content Intake Checklist v1

## 1. 목적

이 문서는 STRAT 홈페이지 고도화를 위해 필요한 회사 실적, 대표/전문가 소개, 프로젝트 사례, 산출물, 문의 전환 콘텐츠를 수집하기 위한 체크리스트다.

목표는 단순히 “내용을 채우는 것”이 아니라, 방문자가 다음 판단을 하게 만드는 것이다.

> 이 회사는 우리 조직의 문제를 이해하고, 실제로 비슷한 문제를 해결해본 곳이다.

## 2. 우선 받아야 할 핵심 자료

### A. 회사 기본 정보

필수:

- 공식 회사명
- 브랜드명 표기
- 설립연도
- 대표자명
- 주소
- 대표 이메일
- 대표 전화번호
- 사업자 정보 공개 여부
- 회사 소개 3줄 버전
- 회사 소개 긴 버전

있으면 좋은 자료:

- 회사 로고 원본
- 브랜드 컬러
- 기존 회사소개서
- 제안서 표지/목차 샘플
- 기존 홈페이지 문구

### B. 회사 포지셔닝

답변 필요:

- STRAT이 가장 잘하는 컨설팅 분야는 무엇인가?
- 경쟁 컨설팅사와 가장 다른 점은 무엇인가?
- 주 고객은 공공기관 중심인가, 민간기업도 중요한가?
- 가장 수주하고 싶은 프로젝트 유형은 무엇인가?
- 피하고 싶은 프로젝트 유형은 무엇인가?
- 고객이 STRAT을 선택하는 가장 큰 이유는 무엇인가?

추천 답변 형식:

```text
STRAT은 [고객 유형]의 [핵심 문제]를 [방법론/강점]으로 해결해 [결과물]까지 만드는 컨설팅사입니다.
```

### C. 프로젝트 실적 목록

필수 컬럼:

- 연도
- 고객명 또는 익명 고객 유형
- 공공/민간 구분
- 산업/기관 유형
- 프로젝트명
- 핵심 과제
- 수행 범위
- 주요 산출물
- 공개 가능 여부
- 홈페이지 노출 가능 수준

노출 가능 수준:

- 고객명 공개 가능
- 고객명 비공개, 기관 유형 공개 가능
- 프로젝트명 일부 공개 가능
- 완전 익명 사례만 가능

### D. 대표 사례 6개

홈페이지에서는 전체 실적보다 대표 사례 6개가 중요하다.

각 사례별로 필요한 내용:

- 고객 유형
- 프로젝트 배경
- 고객이 겪던 문제
- STRAT의 접근 방식
- 핵심 분석/진단 포인트
- 주요 산출물
- 프로젝트 이후 변화 또는 기대효과
- 관련 솔루션 클러스터
- 공개 가능 문장

사례 작성 형식:

```text
고객 유형:
문제:
접근:
산출물:
효과:
관련 솔루션:
공개 가능 여부:
```

### E. 전문가/대표 프로필

필수:

- 이름
- 직함
- 전문 분야
- 핵심 경력
- 대표 프로젝트
- 학력/자격/수상
- 한 줄 소개
- 사진 사용 여부

홈페이지용 프로필은 이력서처럼 길게 쓰지 않는다.

좋은 형식:

```text
[이름]은 공공기관 중장기전략, 조직진단, 성과관리체계 설계를 중심으로 복잡한 경영문제를 실행 가능한 의사결정안으로 구조화해온 전략컨설턴트입니다.
```

### F. 산출물 샘플

가능한 범위에서 필요:

- 전략체계도 샘플
- KPI 맵 샘플
- R&R 매트릭스 샘플
- 조직진단 리포트 목차
- 성과관리 운영안 목차
- 직무기술서 샘플
- 실행 로드맵 샘플
- 최종보고서 목차
- 발표자료 샘플

주의:

- 실제 고객명, 내부 수치, 민감정보는 익명화한다.
- 원본을 그대로 공개하지 않아도 된다.
- 홈페이지에는 “산출물 유형”과 “일부 흐림 처리된 미리보기”만 있어도 강력하다.

### G. 고객 신뢰 요소

수집 항목:

- 고객사/기관 로고 공개 가능 여부
- 총 프로젝트 수
- 공공기관 프로젝트 수
- 주요 산업/분야
- 반복 의뢰 고객 여부
- 파트너/협력 네트워크
- 언론/강의/세미나/자문 이력
- 인증/수상/등록 사항

숫자형 신뢰 문구 후보:

- `공공기관 및 기업 전략 프로젝트 경험`
- `전략·조직·성과·인사 연계 컨설팅`
- `과제 정의부터 최종보고서까지 실행 중심 수행`

정확한 숫자는 자료 확인 후 넣는다.

### H. 인사이트 콘텐츠

초기 콘텐츠 후보:

- 공공기관 중장기전략 착수 전 체크리스트
- 조직진단 전 확인해야 할 질문
- KPI가 전략과 분리되는 이유
- R&R 재정립이 조직개편보다 먼저 필요한 경우
- 성과평가제도가 성과를 만들지 못하는 이유
- 경영평가 대응을 전략체계와 연결하는 법
- AI/DT 전환 전략 수립 전 정리할 질문

각 콘텐츠에 필요한 것:

- 대상 독자
- 독자가 가진 문제
- 핵심 메시지
- STRAT의 관점
- 상담 CTA

### I. 문의 전환 정보

결정 필요:

- 문의 받을 이메일
- 문의 받을 전화번호
- 상담 가능 시간
- 1차 상담 무료 여부
- 상담 방식: 전화/화상/방문/이메일
- 문의 후 응답 시간
- 파일 업로드 필요 여부

권장 CTA:

- `과제 범위 검토받기`
- `진단 결과 기반 상담받기`
- `공공기관 전략상담 예약`
- `유사 사례 확인하고 상담하기`

## 3. STRAT Solution Finder에 필요한 실제 콘텐츠

### 조직 유형별 추천 문구

공공기관:

- 중장기전략
- 경영평가 대응
- 조직진단
- 성과관리체계
- ESG/사회적가치

민간기업:

- 성장전략
- 조직운영체계
- KPI/성과관리
- 신사업전략
- AI/DT 전환

필요 자료:

- 실제 STRAT이 강한 조직 유형
- 각 유형별 대표 프로젝트
- 각 유형별 상담으로 유도할 핵심 메시지

### 솔루션 클러스터별 필요 콘텐츠

1. Strategy Alignment
   - 대표 실적
   - 산출물 샘플
   - 고객 문제 문장

2. KPI & Performance System
   - KPI/성과관리 프로젝트 사례
   - 평가제도 개선 사례
   - 성과관리 산출물

3. Organization & R&R
   - 조직진단 사례
   - R&R/위임전결 사례
   - 조직개편 산출물

4. Work Process & Collaboration
   - 프로세스 개선 사례
   - 협업/조직문화 사례

5. HR, Job & Capability
   - 직무분석
   - 역량모델링
   - 인사제도 개선

6. ESG & Social Value
   - ESG/사회적가치 프로젝트
   - 관련 성과지표 설계 사례

7. AI/DT Transformation
   - AI/DT 전략 또는 업무혁신 사례
   - 데이터/운영체계 개선 사례

## 4. 자료 제공 우선순위

### 1순위

- 회사소개서
- 전체 프로젝트 실적 목록
- 대표 사례 6개
- 대표/전문가 프로필
- 로고/브랜드 파일

### 2순위

- 산출물 샘플
- 고객 로고/기관명 공개 가능 목록
- 기존 제안서/보고서 목차
- 현재 홈페이지 문구 중 유지할 내용

### 3순위

- 인사이트 글 초안
- 세미나/강의 자료
- 언론/외부 활동
- 고객 후기 또는 내부 피드백

## 5. 바로 보내주면 좋은 파일 형태

가능한 파일:

- 회사소개서 PDF/PPT
- 실적 목록 Excel
- 제안서/보고서 샘플 PDF/PPT
- 대표 프로필 Word/PPT/PDF
- 로고 PNG/SVG/AI
- 기존 홈페이지 텍스트

파일이 민감하면:

- 고객명 삭제
- 금액 삭제
- 내부 수치 삭제
- 일부 페이지만 제공
- 스크린샷 제공

## 6. 다음 제작 순서

자료를 받으면 다음 순서로 진행한다.

1. 실적/사례 분류
2. 공개 가능 콘텐츠와 비공개 콘텐츠 구분
3. 대표 사례 6개 선정
4. 서비스 7대 클러스터별 실제 콘텐츠 매핑
5. 홈페이지 카피 고도화
6. Solution Finder 결과 문구 고도화
7. HTML 프로토타입에 실제 콘텐츠 반영
8. 문의 전환 흐름 보강
9. 최종 리뉴얼 기획서 작성
