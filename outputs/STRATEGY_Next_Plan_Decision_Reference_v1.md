# STRATEGY Next Plan / Decision Gate / Reference v1

작성일: 2026-07-02

## 1. 다음 계획

현재 MVP는 다음 흐름까지 구현되어 있다.

```text
문제 후킹
-> Issue Finder
-> 진단 결과
-> 관련 대표실적 proof panel
-> 문의
-> 이벤트 학습
-> 관리자 analytics summary
```

다음 개발은 "더 많은 기능"보다 "문의 전환을 높이는 판단 지점"에 집중해야 한다.

## 2. Sprint 05 제안: Conversion Journey View

목표:

관리자가 단순 이벤트 목록이 아니라, 방문자가 어떤 경로로 문의 의사에 가까워졌는지 볼 수 있게 한다.

범위:

- 세션별 최근 여정 요약
- `issue_finder_start -> issue_result_view -> proof_jump_click -> contact_form_start -> contact_form_submit`
- 관리자 이벤트 탭에서 세션별 journey block 표시

건드리지 않을 것:

- DB/schema
- auth
- payment
- 외부 분석 라이브러리

현재 DB에는 이미 이벤트가 저장되므로, 1차 구현은 기존 `analytics_events` 데이터를 그룹핑해서 보여주는 방식으로 충분하다.

## 3. Sprint 06 제안: Flagship Case Detail

목표:

대표실적 6개를 한 화면 proof panel에서 끝내지 않고, 각각 1페이지짜리 상담 전환 자산으로 만든다.

대상:

- 신용보증기금 뉴비전 수립

- 평창 올림픽 레거시 활용전략
- 한국도로공사 미래사업/UAM 대응전략
- UAM Team Korea 대응전략

필요 콘텐츠:

- 문제 배경
- 왜 어려운 과제였는가
- STRATEGY 접근
- 산출물
- 비슷한 문제가 있는 조직을 위한 질문
- 문의 CTA

## 4. Sprint 07 제안: CEO Weekly Learning Report

목표:

홈페이지가 스스로 "이번 주 무엇을 배웠는지" 요약하게 한다.

출력:

- 가장 많이 선택된 문제유형
- 가장 많이 클릭된 대표실적
- 문의 직전 경로
- 다음 실험 후보 3개

1차는 관리자 화면에서 생성 버튼 없이 자동 요약 블록으로 충분하다.

## 5. 지속하기 전에 대표 판단이 필요한 부분

### A. 기준 레퍼런스 선택

내 추천은 `Linear` 단 하나다.

Reference:

```text
https://linear.app/
```

선정 이유:

- 정적인 소개보다 제품 운영 흐름을 보여준다.
- 카피가 짧고 명확하다.
- UI가 고급스럽지만 장식 과잉이 아니다.
- AI 시대성을 자연스럽게 제품 철학에 통합한다.
- "계획 -> 실행 -> 리뷰 -> 모니터링" 흐름이 STRATEGY의 GitHub식 성장 홈페이지 운영철학과 맞다.

STRATEGY에 적용할 방식:

```text
Linear: product development system
STRATEGY: problem-solving strategy system
```

따라서 STRATEGY 홈페이지도 단순 컨설팅 소개가 아니라, 문제를 구조화하고 실행으로 옮기는 시스템처럼 보여야 한다.

### B. 톤 선택

대표 판단 필요:

```text
1. 더 공공기관/정책 컨설팅 신뢰형
2. 더 스타트업/그로스/AI 시대형
3. 둘을 결합하되 첫 화면은 혁신형, 하단은 신뢰형
```

내 추천:

```text
3. 첫 화면은 혁신형, proof와 people은 신뢰형
```

### C. 대표실적 공개 수위

대표 판단 필요:

```text
기관명과 프로젝트명을 그대로 노출할 것인가?
일부는 익명화할 것인가?
```

내 추천:

이미 공개 가능한 실적은 그대로 쓰고, 민감한 산출물/내부 이슈/성과 수치는 익명화한다.

### D. 문의 전환 방식

대표 판단 필요:

```text
1. 바로 문의 폼
2. 30분 전략상담 예약
3. 진단 결과 기반 상담 요청
```

내 추천:

```text
3. 진단 결과 기반 상담 요청
```

이 사이트의 차별점은 "상담 전에 문제를 짚어주는 경험"이기 때문이다.

## 6. 딱 하나의 최고 레퍼런스

`Linear`를 기준으로 삼는다.

벤치마킹할 것:

- 단문 카피

- 운영 흐름을 시각화하는 정보 설계
- 고급스럽지만 빠른 퍼블리싱 감각
- AI/Agent 시대성을 자연스럽게 녹이는 방식

베끼지 말 것:

- 개발자 도구 톤을 그대로 가져오는 것
- 너무 어두운 SaaS 느낌
- 컨설팅 신뢰 레이어를 약화시키는 과한 제품화
