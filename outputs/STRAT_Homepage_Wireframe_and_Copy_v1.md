# STRAT Homepage Wireframe and Copy v1

## 1. Page Objective

홈페이지의 목적은 STRAT을 설명하는 것이 아니라 방문자를 Solution Finder로 진입시키고, 문제를 자각하게 만든 뒤 상담으로 연결하는 것이다.

Primary conversion:

- Solution Finder 시작

Secondary conversion:

- 사례 보기
- 과제 범위 검토 요청
- 상담 신청

## 3. Hero Copy Options

### Option A. Direct Diagnostic

Headline:

> 지금 조직에 필요한 전략 솔루션을 1분 안에 좁혀보세요.

Subcopy:

> STRAT은 공공기관과 기업의 전략, 조직, 성과, 인사, ESG, AI/DT 과제를 진단하고 실행 가능한 의사결정안으로 설계합니다.

CTA:

- 전략진단 시작
- 사례 먼저 보기

Best use:

- MVP 기본안

### Option B. Problem Hook

Headline:

> 전략은 있는데 실행체계가 따라오지 않는다면, 병목부터 진단하세요.

Subcopy:

> KPI, R&R, 조직구조, 성과평가가 서로 연결되지 않을 때 전략은 실행되지 않습니다. STRAT이 현재 병목과 우선 솔루션을 좁혀드립니다.

CTA:

- 실행 병목 진단하기
- 관련 사례 보기

Best use:

- 공공기관/조직진단 타깃

### Option C. Public Sector

Headline:

> 중장기전략, 조직진단, 경영평가. 지금 우선 해결해야 할 과제를 찾아보세요.

Subcopy:

> 공공기관의 전략은 보고서가 아니라 평가, 조직, 성과, 실행체계로 연결되어야 합니다. 현재 과제를 선택하면 필요한 컨설팅 방향을 제안합니다.

CTA:

- 공공기관 전략진단 시작
- 경영평가 대응 사례 보기

Best use:

- 공공기관 캠페인 랜딩

### Option D. Premium Consulting

Headline:

> 복잡한 경영문제를 실행 가능한 전략, 조직, 성과체계로 바꿉니다.

Subcopy:

> STRAT은 문제 정의부터 전략 설계, 조직 운영체계, KPI, 실행 로드맵까지 의사결정에 필요한 산출물을 만듭니다.

CTA:

- 과제 범위 검토받기
- Method 보기

Best use:

- 브랜드 신뢰 중심 페이지

## 4. Home Section Blueprint

### Section 1. Hero with Solution Finder

Goal:

- 첫 행동 유도

Components:

- Headline
- Subcopy
- CTA
- Mini Solution Finder
- Trust indicators

Success metric:

- Hero CTA click rate
- Finder start rate

### Section 2. Problem Mirror

Goal:

- 방문자가 자기 문제를 인식하게 한다.

Headline:

> 이런 신호가 있다면 전략보다 실행 연결부를 먼저 봐야 합니다.

Cards:

1. 전략은 있지만 부서별 실행과제가 흐릿하다.
2. KPI가 전략과 연결되지 않는다.
3. 조직개편이 반복되지만 R&R 혼선은 줄지 않는다.
4. 평가제도가 성과 향상보다 불만을 만든다.
5. 인사배치와 직무역량 기준이 모호하다.
6. ESG/AI 전환 과제가 실행 로드맵으로 정리되지 않았다.

CTA:

- 내 조직의 병목 확인하기

### Section 3. Solution Finder Preview

Goal:

- 진단 경험의 가치를 보여준다.

Headline:

> 3단계로 필요한 솔루션을 좁힙니다.

Steps:

1. 조직 유형 선택
2. 현재 고민 선택
3. 핵심 문항 응답
4. 추천 솔루션 확인

Result preview:

> 현재 조직은 “전략은 있으나 KPI, R&R, 성과평가로 이어지는 실행 연결부가 약한 상태”로 보입니다.

CTA:

- 1분 전략진단 시작

### Section 4. Solution Clusters

Goal:

- STRAT의 서비스 체계를 문제 해결 방식으로 보여준다.

Headline:

> STRAT은 문제를 7개 솔루션 체계로 좁혀갑니다.

Clusters:

1. Strategy Alignment
2. KPI & Performance System
3. Organization & R&R
4. Work Process & Collaboration
5. HR, Job & Capability
6. ESG & Social Value
7. AI/DT Transformation

Each card:

- Problem
- Approach
- Deliverable
- CTA

### Section 5. Cases

- 신뢰 형성

Headline:

- 문제

## 5. Page Interaction Notes

### Sticky CTA

Desktop:

- Header right: `1분 진단`

Mobile:

- Bottom sticky button: `전략진단 시작`

### Finder Modal vs Inline

MVP recommendation:

- Hero에서는 inline
- 다른 페이지에서는 modal

Reason:

- 첫 화면에서는 제품처럼 보여야 한다.
- 사례/인사이트 페이지에서는 읽기 흐름을 방해하지 않고 CTA로 열리면 된다.

### Result Capture

결과 화면 전 연락처를 요구하지 않는다.

권장 흐름:

1. 진단 완료
2. 결과 일부 공개
3. “결과 요약 받기” 또는 “상담받기”에서 연락처 요청

이유:

- 사용자가 먼저 가치를 경험해야 전환 저항이 낮아진다.

## 6. Contact Conversion Copy

### General

> 진단 결과를 바탕으로 귀 조직에 필요한 과제 범위와 착수 방향을 검토해드립니다.

### Public Sector

> 중장기전략, 경영평가, 조직진단 과제의 범위와 산출물을 상담 전 검토해드립니다.

### Performance

> KPI, 성과평가, 모니터링 체계 중 어디서 병목이 생기는지 진단 결과를 바탕으로 검토합니다.

### Organization

> 조직구조, R&R, 인력배분 이슈를 현재 진단 결과와 연결해 상담합니다.

## 7. Design Tone

Keywords:

- precise
- strategic
- calm
- fast
- diagnostic
- evidence-based
- premium but not old-fashioned

Avoid:

- 장식적인 컨설팅 이미지
- 추상적인 슬로건만 있는 첫 화면
- 서비스 나열형 카드 과다
- 지나치게 무거운 공공기관 보고서 톤
- 스타트업처럼 가벼운 과장 표현

Recommended visual language:

- 진단 패널
- 전략 맵
- KPI 연결선
- 조직/R&R 매트릭스
- 결과 카드
- 데이터 기반 신뢰 지표
