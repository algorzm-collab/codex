# Sprint 05 — Conversion Journey View (Admin)

## 목적
- 이벤트 로그를 스키마 변경 없이 `session_id` 기반 세션 단위로 묶어 상담 전환 흐름을 한 화면에서 확인한다.
- 문의 유도 성과 판단 근거를 “총 이벤트/문제유형/타깃” 집계에서 “여정 전환 상태”로 확장한다.

## 구현 범위 (이번 스텝)
- Admin Events 탭 상단에 Conversion Journey 요약 카드 추가
- 동일 세션 내 이벤트를 시간순 정렬해 패스(Path) 시각화
- Converted 판정 키워드(`inquiry`, `contact`, `submit`) 기반 자동 집계
- 기존 API 스키마를 변경하지 않고 클라이언트 집계 방식으로 렌더링

## 다음 판단 포인트 (내부 결정용)
- KPI 후보
  - 변환된 세션 비율
  - 핵심 페이지별 초깃값 진입 → 문의완료 단계의 이탈율
- 질문 1개 남김 (우선순위)
  - `converted` 판정 기준을 현재 키워드 기반으로 유지할지, 향후 DB 필드로 명시 이벤트를 늘릴지?

## 현재 리스크
- 이벤트명 표준이 완전하지 않으면 Converted 판정이 보수적으로 측정될 수 있음
- 세션 식별자가 누락된 로그는 `no_session_id` 그룹으로 분리됨

## 다음 액션
- 운영 중 실 로그 3~7일 집계 후 전환 임계값을 기준으로 경고 메시지 및 액션 탭 확장
