# Orchestration + Harness Model

## Purpose

ChatGPT Projects, Library, Apps, Codex, and GitHub are surfaces.

They are useful interfaces, but they are not enough by themselves.

The stronger model is:

```text
CEO Cockpit
  ↓
Portfolio Orchestrator
  ↓
Project Cells
  ↓
Mission Queue
  ↓
Agent Router
  ↓
Harness Gate
  ↓
Execution Surface
  ↓
Evidence Package
  ↓
Report / Learning
```

---

## Key Idea

Do not let each project become an isolated chat.

Each project should be a cell in one portfolio orchestra.

Each cell can use ChatGPT Project as its workspace, but the portfolio orchestra decides:

- priority
- next mission
- owner agent
- required context
- success checks
- handoff rules
- report format

---

## Project Cell

A Project Cell is the operating unit for one business/project stream.

Each Project Cell has:

- ChatGPT Project
- GitHub repository or workspace
- Mission Queue
- context pack
- reusable library assets
- Codex micro-task stream
- harness checks
- latest report

---

## Harness Gate

No work is considered complete merely because an agent says it is complete.

Every mission should pass an appropriate harness.

Harness types:

- context harness: was the minimum correct context selected?
- scope harness: was the task small enough?
- output harness: did the output meet success criteria?
- evidence harness: is there proof, file diff, report, test, or artifact?
- handoff harness: can the next agent continue without asking the CEO?
- executive harness: can the CEO understand status in one glance?

---

## Orchestrator Responsibilities

The Portfolio Orchestrator owns:

- cross-project priority
- mission scheduling
- agent routing
- capacity-aware task sizing
- dashboard summaries
- risk escalation
- leverage radar intake
- project health scoring

---

## Why This Is Better Than One Chat

A single Q&A chat is good for one thread.

A portfolio of 10+ projects needs:

- memory separation
- status aggregation
- task queues
- reusable assets
- project-specific context
- cross-project prioritization
- evidence and review gates

Therefore:

- ChatGPT Project = project room
- Portfolio Orchestrator = conductor
- Harness Gate = quality control
- GitHub = record and execution state
- Codex = micro-task player

---

## Golden Rule

Native interfaces are surfaces.

The orchestra is the operating system.

The harness is the truth test.
